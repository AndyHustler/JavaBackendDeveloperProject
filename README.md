## Проект по второму блоку обучения Java Backend Developer

По техническим причинам - не смог поднять docker на своей ОС Windows - в качестве БД используется com.h2database

[Swagger UI с комментариями к ендпоинтам](http://localhost:8080/swagger-ui/index.html)

### Функционал пользователя
   1. Работа со своими задачами:
        - просмотр списка своих задач: __/app/task-api/myTasks__
        - создание новых задач: __/app/task-api/create__
        - изменение своих задач: __/app/task-api/update__
        - удаление своих задач: __/app/task-api/delete__
        - изменение статуса задачи (например, «запланирована», «в работе», «сделана»): __/app/task-api/update/{id}/status/{status}__
        - привязка задачи к группе задач и изменение группы: __/app/task-api/update/{id}/grop/{grop}__
   2. Работа с группами задач:
        - просмотр списка своих групп задач: __/app/task-group-api/curent-user-groups-list__
        - создание своих групп задач: __/app/task-group-api/create__
        - переименование своих групп задач: __/app/task-group-api/update__
        - удаление своих групп задач: __/app/task-group-api/delete__
   3. Регистрация с указанием почты и пароля: __/auth__
   4. Авторизация с указанием почты и пароля уже зарегистрированного пользователя: __/signin__
   		- в заголовке возвращается http-only cookie с токеном доступа и рефреш токеном
   		- для демонстрации эти токены возвращаются в теле ответа
   5. Обновление токена: __/refreshtoken__

### Функционал администратора
   1. Просмотр списка пользователей: __/app/user-api/users-list__
   2. Просмотр всех задач в системе: __/app/task-api/tasks-list__
   3. Просмотр всех групп задач: __/app/task-group-api/groups-list__
   4. Просмотр задач выбранного пользователя: __/app/task-api/user/{id}__
   5. Просмотр простой агрегированной статистики(например, количество задач по статусам или по группам задач).
